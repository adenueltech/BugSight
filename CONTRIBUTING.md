# Contributing to BugSight

We love contributions! Here's how you can help make BugSight better.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/bugsight.git`
3. Create a branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m 'Add amazing feature'`
7. Push: `git push origin feature/amazing-feature`
8. Open a Pull Request

## Development Setup

See SETUP.md for detailed setup instructions.

## Code Style

- Use TypeScript for type safety
- Follow existing code formatting
- Add comments for complex logic
- Keep functions small and focused
- Use meaningful variable names

## Testing

- Test all new features manually
- Ensure web app works in dark/light mode
- Test VSCode extension in development host
- Verify API integration works

## Bug Reports

Found a bug? Open an issue with:
- Clear description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Environment details

## Feature Requests

Have an idea? Open an issue with:
- Problem you're solving
- Proposed solution
- Alternative approaches
- Mockups/examples if applicable

## Questions?

Open a discussion or reach out to the maintainers.
